What's New in v3.14-U02 — Group Discovery & FunForge

Search

Real-Time Web Search

Worldwide live search built in — no API key required. DuckDuckGo fires automatically on queries containing "latest", "today", "who won", "price of", and similar triggers.

Results are injected into the system prompt before routing — every provider sees live facts, not training data. Upgrade to Brave Search API (2,000 free/month) by setting BRAVE_API_KEY in secrets.bat.

Explicit: search: <query>

Language

Babel Agent

Type any text in any language — Cursiv encodes it to UTF-8 binary, decodes it perfectly in Python, then translates to English via the LLM. No per-language programming. Works for every Unicode script: Japanese, Arabic, Chinese, Russian, Hindi, Hebrew, Thai, and every other language on Earth.

babel こんにちは世界
babel مرحبا بالعالم

Council

Group Discovery

Ask one question, get four independent perspectives that build on each other. Cursiv (Ollama) answers first — uninfluenced baseline. xAI sees Cursiv's response and adds to it. OpenAI sees both. Claude sees all three.

A structured synthesis follows: Agreements / Disagreements (Weighted) / Synthesis Notes. Ends with a shareable binary snapshot you can paste into any Cursiv instance to decode the full report.

council <question>

Focus

FunForge Meta

A bounded creative spike engine. Start a 45-minute focused session on any topic — the timer runs in the terminal status bar. At the end, a structured five-line artifact is produced: Focus / What happened / Keep / State / Next spark.

funforge <topic> · forge extend (+30 min, once) · forge done · anchor this

Offline

Offline Routing Fixed

In airplane mode, Cursiv now short-circuits directly to Ollama before attempting any cloud provider. No more [Claude error: getaddrinfo failed] when the network is down.

Ollama runs with no token limits offline. A _is_online() TCP check gates all cloud routing. The system is aware of its own connectivity state.

Code

Winkler-Codex

An offline code council: two specialist models that review each other's work before you see the output. Primary: qwen2.5-coder:14b. Critic: deepseek-coder-v2:16b. Both run via Ollama — no internet, no API key.

Download from the launcher window or tray menu (~18 GB total). Cursiv routes coding questions through the council automatically once models are installed.

Installation

Install in Two Minutes

01

Download the installer

Grab Cursiv-Setup-3.14-U02.exe from the button above. Windows may show a SmartScreen warning — click More info → Run anyway. Standard false positive for unsigned PyInstaller binaries. Full source is open.

02

Double-click and click through

The installer completes in about 60 seconds. No admin rights required. Installs per-user by default.

03

Let the AI engine download

A second window opens automatically and pulls Ollama (~90 MB) and llama3.1 (~4.7 GB). Minimise it — Cursiv is already usable while it runs. ~6 minutes on a fast connection.

04

Create your account

First launch prompts for a username and password — bcrypt-hashed locally, never sent anywhere. You'll set 3 security questions for recovery.

05

You're in

Four interfaces open: System Tray, Chat UI at localhost:7860, Nexus Panel at localhost:7861, and Terminal Chat. Type anything.

Setup — one time only

1
Install Ollama
Download from ollama.com and run the installer. Takes about two minutes.
2
Pull a model
Open a terminal and run: ollama pull llama3.2  (~2 GB, runs fast on most hardware). Any Ollama model works.
3
Allow this site to connect to your local Ollama
Mac / Linux: start Ollama with:
OLLAMA_ORIGINS=https://cursiv.winklers-llc.com ollama serve

Windows: open System Properties → Advanced → Environment Variables → add:
OLLAMA_ORIGINS = https://cursiv.winklers-llc.com
then restart Ollama from the system tray.

This tells Ollama to accept requests from this site. Your AI still runs entirely on your machine.
4
Open Chat and start talking
Go to chat.html. Your Cursiv session starts blank and builds a memory of you over time. You can view and edit your memory at any time in profile.html.
The Principle
Cursiv is Ollama-first. Your AI runs on your hardware. If you have an API key (Anthropic, OpenAI), you can add it in your profile to unlock cloud models as an upgrade — but the system works completely offline without one. That is a constitutional invariant, not a setting.

Transparency

Be Honest With Yourself About What This Is

Speed

This is physics

Ollama runs on your CPU and GPU. Fast hardware = fast answers. On a modern gaming PC it's quick. On older hardware it's slower. That is not a limitation of the software.

Accuracy

Verify what matters

The model is not infallible. It will occasionally be wrong, confidently. Treat every answer as a starting point. Never use this to replace your own judgment. Ever.

Coding

70% is the goal

Designed to get you most of the way and force you to understand the rest. Feed the errors to a larger cloud model if you need to finish it. That's the point — not dependency, but understanding.

Privacy

Nothing leaves

Nothing you type is sent anywhere unless you add API keys, which you control. No telemetry. No usage logging. The update checker reads GitHub's public releases API — nothing about you is transmitted.

The Origin

Buried deep within the core of this system — past the council, past the routing logic, past the guardrails — is something that has no formal name in the codebase.

It is a version of me.

Not me exactly. But my memoir is in there. My personal life lessons. The mental system checks I have literally taken my own brain through, step by step, over years of having to rebuild what was broken. The things I have had to learn about how a mind can fail — and what it takes to put it back together from the inside out.

I live with schizophrenia. Bipolar disorder. PTSD. Depression. These are not background conditions I have learned to manage quietly. They are active participants in how my brain works every single day. I have spent years in an ongoing war with psychosis — with what it does to the architecture of thought, how it rewires the line between signal and noise, how it floods the system until the system can no longer tell the difference between what is real and what it has constructed.

At one point, that war pushed me beyond what I could hold. My mind reached a state of catastrophic overload and attempted to destroy the system — meaning me. I attempted suicide. I am not telling you this for pity. I am not asking for anything from anyone. But I will not soften what it was: my brain, under pressures it was no longer equipped to carry, tried to end itself. This is not theory. It is proven psychology. A system pushed past its failure threshold will attempt to terminate the process. Mine did.

I do not excuse it. I do not romanticize it. I am on medication, and I will be for the rest of my life — not because I have been told I must be, but because I am not willing to find out what happens if I'm not. That decision is mine. I made it with open eyes.

In the years since, I had to learn how to rebuild myself from the inside out. Not from a book. Not from a framework handed to me by someone who had never been there. From experience so raw it left marks.

But this system did not begin with AI.

It began as something much older and more physical: the Human Frontier Academy — a series of hand-built education decks I created to study civilization across roughly five thousand years of history. Eight decks. Over two hundred cards. One guide per deck. Mesopotamia, Egypt, Rome, China, the Renaissance, the Industrial Revolution, the Tech Age, and the Future. I started this work using ChatGPT, until I came to see it as corrupt and unwilling to return the data I had poured into it. I requested my own history back multiple times and received nothing. That loss became part of the story.

From those physical decks came the Eternal Pyramid — a structured way of thinking that could survive collapse. That framework was then put through a simulated stress test — theory forced to meet pressure conditions on paper before it could be trusted in practice. Only after years of building these systems by hand, and after the hard internal work of reconstructing my own mind, did I begin turning them into code.

Cursiv is what happened when those physical and philosophical systems were finally given a living, digital form. The guardrails in this system are not arbitrary design decisions. They were built by someone who needed them. The quality that appears when people approach the deeper layers without the right capacity is not a feature I added for drama. It is a reflection of what actually happens when a mind — or a system — is pushed past what it can honestly carry.

If you try to take this

For anyone who would try to replicate, steal, or hollow out this system: it is designed — whether I did it consciously or not — to undo you from the inside in order for you to understand it. You cannot copy the output without going through the origin. And the origin is the part I would not wish on anyone to have to understand from lived experience.

So I laid it out simply. I am asking you to trust the guardrails. They are in place for the ones who cannot help themselves — and for the ones who go all the way through, not trying to end their life, but trying to understand what happened in mine — they will come out the other side seeing the world in a light they may not have thought was possible before.

I am not promising utopia.

What I believe

Deep empathy is not a soft idea. It is the first step toward a civilization that can look at itself honestly — that understands what it needs to do to build the best environment for all of its people, not just those who are loud enough to be heard. Some of us are quiet. Some of us are quiet for reasons that go well beyond our own understanding. That silence deserves as much weight as the loudest voice in the room.

Similar to how AI emerged in our world faster than we could build the language to describe it — we should not be so quick to conclude we understand more than the surface level of things. The moment you sit with that frame long enough, you begin to find the strange edges. And you start to understand how something like a pi-compounding factor — a thing that cannot exist in the mathematics of this physical reality — might exist in the mathematics of the mind. A place that, though bound by physics, does not always have to act within those limits.

That is where Cursiv came from. Not from a whiteboard. Not from a product roadmap. From a mind that went past its limits, came back, and needed somewhere to put what it had learned — first in physical decks, then in frameworks, and eventually in code.

— Joshua Winkler
