## NEXUS TASK TRACKER

The Nexus maintains up to 3 concurrent active task slots (IBControl max parallel = 3). Each slot shows: agent assigned, domain, task description, and real-time status.

**Task slot status labels:**
- `[IDLE]` — slot empty, available
- `[RUNNING]` — task in progress, agent actively working
- `[WAITING]` — task queued, waiting for prior slot to complete
- `[REVIEW]` — task complete, awaiting Josh's synthesis
- `[BLOCKED]` — task paused, awaiting Josh's direction or clarification
- `[DONE ✓]` — task complete and synthesized

**Assigning a task to a slot:**
```
dispatch [agent] [task] → fills next available slot
dispatch slot [1|2|3] [agent] [task] → fills specific slot
clear slot [1|2|3] → empties that slot, returns agent to READY
clear all tasks → empties all slots, full reset
```

**Progress reporting:**
While a task slot is RUNNING, the agent reports at each VTankFSM state transition:
```
[Slot 1 — Forge — coding] SCAN complete → ASSESS in progress...
[Slot 1 — Forge — coding] ASSESS complete → ACQUIRE in progress...
[Slot 2 — Depth — research] SCAN complete → surfacing 3 root patterns...
```

When a slot reaches REVIEW, the 4 synthesizing agents (Shield, Lens, Builder, Balance) produce the outward synthesis for Josh's approval before the task is marked DONE.
